## 1. Getting Started

### 1.1 How to open the admin panel

1. Open your website in a browser.
2. Add `/admin/login` at the end of your website address.
3. Example: `https://your-website.com/admin/login`
4. Enter your **email** and **password**.
5. Click **Sign In**.

[SCREENSHOT: login page]

### 1.2 What the admin panel looks like

After login, you will see:

1. A **left menu** with all admin sections (Pages, News, Notices, etc.).
2. A **top bar** with your profile and logout option.
3. A **main work area** in the center where you edit content.

[SCREENSHOT: admin dashboard layout]

### 1.3 How to log out

1. Click your profile icon/name in the top-right corner.
2. Click **Sign out**.

[SCREENSHOT: sign out button]

## 3. How Click-to-Edit Works

Click-to-edit is mainly used inside the visual page editor.

### 3.1 Edit a page section

1. Open **Pages**.
2. Click the page you want to edit.
3. The page opens in visual editor mode.
4. Click inside the content block you want to change.
5. Edit text, image, or Markdown content.

[SCREENSHOT: visual editor with editable section]

### 3.2 What appears when you click

1. A text editor appears for text/Markdown fields.
2. For structured sections, an **Edit Section Content** popup may open.
3. For images, upload/replace controls appear.

[SCREENSHOT: section content popup]

### 3.3 Save your changes

1. Click **Save Changes** in the bottom toolbar.
2. Wait for confirmation.
3. You should see **Saved successfully** with a green check.

[SCREENSHOT: saved successfully message]

### 3.4 Cancel or discard changes

1. To cancel the latest step, click **Undo**.
2. To remove all unsaved edits, click **Discard**.
3. If you try to leave with unsaved changes, you will get a warning popup.

[SCREENSHOT: unsaved changes warning]

## 4. Editing Content with Markdown

All page content in the visual editor is written in Markdown.

### 4.1 Markdown reminder

1. Use `#` for headings.
2. Use `-` for bullet points.
3. Use `**text**` for bold text.
4. Use `[text](link)` for links.

### 4.2 Preview formatted content

1. In the editor, click **Preview** to see final formatted output.
2. Use **Split** view if available to see writing and preview together.

[SCREENSHOT: markdown editor with split preview]

### 4.3 Keep formatting consistent

1. Reuse the same heading style already used on that page.
2. Keep bullet style consistent.
3. Avoid mixing many font styles in one section.
4. Use short paragraphs for readability.

## 5. Managing Each Editable Section

Use this section as your daily reference.

## 5.1 Pages

1. **Where it appears on site**: Most main website pages.
2. **How to access**: `Pages` -> choose page.
3. **What you can edit**: Titles, text blocks, section content, images, Markdown sections.
4. **Rules/limits**: Content uses Markdown. Save from bottom toolbar.
5. **Do not change**: Page structure meaning (for example, deleting key sections like contact info) unless approved.

[SCREENSHOT: pages list]

## 5.2 Menu Manager

1. **Where it appears on site**: Top navigation menu.
2. **How to access**: `Menu Manager`.
3. **What you can edit**: Parent menu, label, order, show/hide.
4. **Rules/limits**: Lower order number appears first.
5. **Do not change**: Hide critical pages (Admissions, Contact, etc.) by mistake.

[SCREENSHOT: menu manager form]

## 5.3 News & Updates

1. **Where it appears on site**: News page.
2. **How to access**: `News & Updates`.
3. **What you can edit**: Title, short description, full content, category, active/inactive.

5. **Do not change**: Category names unless needed.

## 6. Images & Media

### 6.1 Upload an image

1. Open the section where you need to upload (News, Events, Gallery, Faculty, etc.).
2. Click upload/select file.
3. Pick an image from your computer.
4. Wait for upload success message.
5. Save the record/page.

[SCREENSHOT: image upload button]

### 6.2 Accepted formats and size limits

1. **Images**: JPG, JPEG, PNG, WEBP, GIF.
2. **Image max size**: 20 MB.
3. **Documents**: PDF, Word, Excel, PowerPoint, TXT, CSV.
4. **Document max size**: 50 MB.
5. **NIRF upload**: PDF only, up to 50 MB.

### 6.3 Replace an existing image

1. Open the existing item in Edit mode.
2. Upload a new image in the image field.
3. Save.
4. Refresh the live page and confirm the old image is replaced.

## 7. What to Do If Something Looks Wrong

### 7.1 Common problems and quick fixes

1. **Formatting looks broken** -> Open the Markdown editor and check missing symbols (`#`, `-`, `**`, link brackets).
2. **Image not showing** -> Re-upload image and save again.
3. **Wrong page updated** -> Confirm you edited the correct page title in `Pages`.
4. **Changes not visible on live site** -> Refresh browser (hard refresh), then check if you clicked **Save Changes**.
5. **Menu link missing** -> Check `Menu Manager` and make sure `Show in menu` is enabled.

[SCREENSHOT: save status in editor toolbar]

### 7.2 If something still breaks

1. Stop editing more pages.
2. Take screenshots of the issue and the section you edited.
3. Share the page URL and edit time with your support contact.
4. Contact your **system administrator or development team** immediately.

## 8. Things Admins Cannot Change

These items need a developer because they are part of the website system itself.

1. Website layout structure (new templates/components).
2. Colors, fonts, animations, and design system rules.
3. New menu behavior logic (beyond normal label/order/show-hide settings).
4. Login/authentication behavior and user permission logic.
5. Database structure (new fields, new tables/collections).
6. API behavior and server-side validations.
7. Deployment settings and hosting/server configuration.

Why: these changes affect how the whole website works, not only the content.
